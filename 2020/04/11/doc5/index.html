5.1

5.1.2覆盖方法

  • 子类中的方法不能直接访问父类中的private域，可以借助共有接口。
  • 子类访问父类中和自己相同签名的方法可以使用super关键字。
  • super和this不是类似的概念，super不是一个对象的引用，不能将super赋值给另一个对象变量，它只是指示编译器调用父类方法的特殊关键字。

5.1.3子类构造器

super 调用了父类的构造器，由于Manager不能访问父类的私有域，所以必须利用父类的构造器对这部分私有域进行初始化，我们可以通过super调用父类的构造器但是必须是第一句。如果超类没有不带参数的构造器并且子类没有显式调用超类的其他构造器，报错。

this关键字两个用途：一是引用隐式参数，二是调用该类其他的构造函数。同样，super也有两个用途，一是调用超类的方法，二是调用超类的构造器。调用构造函数的时候，必须都是第一句。

5.1.6 理解方法调用

下面假设要调用x.f(args),隐式参数x声明为类C的一个对象。调用过程如下：

  1. 编译器查看对象的声明类型和方法名，可能存在多个名字为f但是参数类型不同的函数，例如f(int),f(String)。编译器将会一一列举所有C类中方法名为f的方法和其超类中属性为public且名为f的方法。
  2. 然后查看调用时候提供的参数类型，如果所有名为f的方法中存在一个与提供的参数类型完全匹配，就选这个方法。这个过程成为重载解析。
  3. 如果是private,static,final方法，或者构造器，那么编译器可以准确的知道应该调用哪个方法，我们将这种调用方法称为静态绑定。与此对应的是，调用的方法依赖于隐式参数的实际类型，并且在运行时动态绑定。
  4. 当程序运行，并且采用动态绑定调用方法时，假设x的实际类型是D，D是C的子类，如果D类定义了方法f，就直接调用它，否则，将在D类的超类中寻找，以此类推。

在覆盖一个方法的时候，子类方法不能低于超类方法的可见性。

5.1.7阻止继承：final类和方法

final：

  1. 阻止定义子类。
  2. 子类不能覆盖方法。
  3. 域值不能改变。

一个类声明为final，只有其中的方法会自动变成final，域不会。

5.1.8 强制类型转换

  1. 向上转型是多态的体现。
  2. 向下转型不安全的，需要使用instanceof 进行检查之后再进行转换。

5.1.9 抽象类

1.类即使不包含抽象方法也可以声明为抽象类。

5.1.10 受保护访问

  1. 仅对本类可见private
  2. 全部可见public
  3. 本包和全部子类可见protected
  4. 本包可见，不需要修饰符

5.2 Object 所有类的超类

只有基本类型不是对象，所有的数组类型，不管是对象数组还是基本类型的数组都扩展了Object类。
